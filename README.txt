
# Whisper CPU Optimization

**Авторы:** Цыганов Пётр, Павел Спицын, Михаил Замешаев  

---

## Цель проекта

- Сравнить различные подходы к оптимизации инференса моделей Whisper  
- Снизить задержки и потребление памяти  
- Добиться работы на CPU в режиме *real-time* ($RTF < 1$) без существенной потери качества  

---

## Pruning

Проведены эксперименты с **unstructured magnitude pruning**.

Удаление части весов позволило уменьшить RTF, особенно для крупных моделей, при минимальном ухудшении качества.

| metric | small 0.1 | small 0.2 | small 0.3 | medium 0.1 | medium 0.2 | medium 0.3 | large 0.1 | large 0.2 | large 0.3 |
|--------|-----------|-----------|-----------|------------|------------|------------|-----------|-----------|-----------|
| rtf | 0.229630 | 0.226232 | 0.220751 | 0.624964 | 0.638332 | 0.601354 | 1.080871 | 1.034719 | 1.047473 |
| wer | 0.382114 | 0.373984 | 0.382114 | 0.378049 | 0.369919 | 0.349593 | 0.288618 | 0.288618 | 0.280488 |
| cer | 0.241038 | 0.225587 | 0.239802 | 0.257726 | 0.244747 | 0.232386 | 0.192213 | 0.186650 | 0.195303 |

Для large-v3 удалось снизить $RTF$ до ~**1.03**, приблизившись к real-time на CPU.

## Quantization

> Важно: эксперименты с прунингом и квантованием проведены на разных датасетах, поэтому WER/CER напрямую не сравниваются.

Цель: оценить изолированный эффект квантования на производительность.

Успешные конфигурации:

- `faster-whisper | CPU | int8`
- `faster-whisper | CPU | int8_float32`


### Результаты

|------|--------|-------|-----|-----|----------------------|------------------|---------------|
| faster-whisper CPU int8 | ok |  |  | 0.0157 | 190.118 | 146.035 | 209.352 |
| faster-whisper CPU int8_float32 | ok |  |  | 0.0157 | 172.881 | 146.035 | 188.493 |

## Ключевые наблюдения

- Unstructured pruning позволяет уменьшить вычислительную сложность модели  
- Наибольший эффект наблюдается на больших моделях (`medium`, `large`)  
- Квантование даёт значительный прирост к скорости  
- Стоит попробовать комбинированный подход
